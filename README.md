# Club Duel Bot — мультигрупповой бот войны кланов для Telegram

Телеграм-бот на **aiogram 3**: система кланов и дуэлей-«сапёра» на очки
клана. **Мультигрупповой** — бота можно добавить в сколько угодно групп, и в
каждой будет своя независимая война (кланы/очки/дуэли), плюс общемировой топ
кланов среди всех групп сразу.

## Установка и запуск

```bash
python3 -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Токен бота и ID администратора уже вписаны в `config.py` (при желании
вынеси в переменные окружения `BOT_TOKEN` / `ADMIN_ID` — код это уже
поддерживает).

### Первый запуск

Никакой настройки не требуется — просто добавь бота в любую группу и дай
ему права администратора (нужно для закрепления сообщений о дуэлях и для
`/votekick`). Группа автоматически регистрируется и заводит свою войну при
первом использовании любой игровой команды в ней. Команда `/setgroup`
убрана — она больше не нужна.

1. Участники создают кланы командой `/createclan` (спросит название →
   аватар (или «Пропустить») → девиз) и вступают в существующие кланы
   командой `/join` (карусель ⬅️/➡️ — **одно и то же сообщение
   редактируется**, новых сообщений при листании не создаётся). Обе команды
   работают только прямо в группе (у каждой группы свой набор кланов).
2. `/clan` — карточка своего клана: очки, серии побед, участники, лучший
   множитель за бой и т.д. Работает и в группе, и в ЛС боту (если игрок
   состоит в клане ровно одной группы — иначе попросит вызвать прямо в
   нужном чате).
3. Когда бот сам объявит дуэль внутри группы (тегнет двух игроков) — один
   из них жмёт `/minduel`, чтобы открыть выбор количества мин и погнать игру.
4. `/top` — топ кланов ЭТОЙ группы. `/globaltop` — топ-25 кланов среди ВСЕХ
   групп, где есть бот. `/online` — сколько групп и уникальных игроков
   сейчас использует бота.

## Как устроена игра

- Поле 5×5 (25 клеток). Игрок выбирает количество мин (1–6) — чем больше
  мин, тем быстрее растёт множитель выигрыша.
- Оба игрока дуэли играют **на своих отдельных полях параллельно**
  (независимо друг от друга), как и было в ТЗ.
- При открытии безопасной клетки множитель растёт по формуле из ТЗ
  (проверено — совпадает с примерами `x1.01→x1.05→x1.10...` для 1 мины,
  `x1.28→x1.70→x2.30...` для 6 мин).
- Можно нажать «✅ Забрать очки» в любой момент и зафиксировать текущий
  выигрыш, либо продолжать рисковать.
- Если попал на мину — очки **всего клана** умножаются на `0.75`, играть
  дальше нельзя.
- Когда **оба** игрока дуэли закончили свою игру (забрали очки или
  подорвались), бот определяет клан-победитель дуэли, обновляет серию
  побед (`current_win_streak` / `max_win_streak`) и счётчик побед в войне
  кланов (`wars_won`).
- **AFK-защита**: если игрок не делает ход дольше `AFK_TIMEOUT_SECONDS`
  (по умолчанию 3 минуты, см. `config.py`), бот автоматически «забирает»
  его текущий выигрыш за него, чтобы дуэль не зависала — это и есть
  «переход хода при АФК» из ТЗ.

## Портал, прозрачность, банк, новые предметы (последнее обновление)

- **Тактика "Вращайте барабан" 🌀** — на поле появляется клетка-портал; клик
  сбрасывает раунд (множитель сгорает), поле пересоздаётся с тем же числом
  мин, а ВСЕ множители умножаются на 1.5 накопительно (1.5→2.25→3.375→...).
  Переходов не ограничено, попытка при этом не тратится.
- **Предмет "Кубик-нубика" (150 Те)** — та же механика портала на 4 ваших
  следующих раунда, даже если у клана другая тактика. Если тактика клана уже
  "Вращайте барабан" — у купившего игрока будет сразу 2 портала на поле
  (у остальных в клане — по-прежнему 1).
- **Прозрачность** — после каждого завершённого раунда (победа или
  поражение) бот отдельным сообщением показывает, где реально были мины (🔴)
  и порталы (🌀).
- **Порядок применения бонусов к выигрышу** (фиксированный): 1) базовое поле
  → 2) усиление портала → 3) % тактики клана → 4) % недельного модификатора
  → 5) личные предметы игрока (например "Сливы, виноград") — применяются
  последними, после всего остального.
- **"Сливы, виноград" (40 Те)** — 4 раунда: если выигрышный x < 10, с шансом
  50% он поднимается до x10, иначе −30% от выигрыша. При подрыве на мине не
  срабатывает. Раунд считается использованным в любом случае.
- **/bank** — вклад валюты Те под 2% в день сложным процентом, снять можно
  не раньше чем через 2 дня после последнего пополнения. Все условия
  показываются при каждом вызове команды.
- **Смена названия клана** подешевела до 15 Те (было 80).
- **/clan** теперь присылается вместе с аватаркой клана (если она есть) —
  одним сообщением, фото + подпись.

## Мультигрупповая архитектура (важно понимать)

- **Каждая группа — своя независимая война.** Кланы, очки, очередь ходов,
  дуэли, магазин, банк, валюта Те — всё это своё в каждой группе. Данные
  хранятся в `db["chats"][str(chat_id)]` (см. `bot/chat_state.py`).
- **Сезон один общий для всех групп** (30 дней, еженедельные события, годовой
  сброс репутации — таймер общий), но КОНКУРЕНЦИЯ (места, топ/аутсайдер,
  ачивки "1 место в сезоне") считается отдельно внутри каждой группы —
  группы не соревнуются друг с другом напрямую.
- **`/top`** — топ-25 кланов текущей группы. **`/globaltop`** — топ-25 кланов
  среди ВСЕХ групп сразу (агрегируется на лету, ничего отдельно не хранится).
- **`/online`** — сколько групп и сколько УНИКАЛЬНЫХ игроков (dedup по
  user_id, даже если человек в нескольких группах) сейчас используют бота.
- **Команды в ЛС боту** (`/clan`, `/iam`, `/shop`, `/bank`, `/season`) —
  работают, если пользователь состоит в клане РОВНО ОДНОЙ группы (тогда бот
  сам понимает, о какой группе речь). Если групп несколько или ни одной —
  бот попросит вызвать команду прямо в нужной группе. Команды, создающие
  что-то новое (`/createclan`, `/join`, дуэли) — только внутри группы,
  в ЛС не работают в принципе.
- **`/setgroup` и `/forceduel` убраны** — они были нужны только в старой
  однo-групповой архитектуре. Регистрация группы теперь полностью
  автоматическая.
- **Дуэли теперь каждые ~4 часа (±40 минут)** вместо раза в день, у каждой
  группы свой независимый таймер.
- **Существующие данные не теряются при обновлении** — при первом запуске
  новой версии бот сам мигрирует старую (одногрупповую) базу в новую схему,
  привязывая её к тому чату, который был назначен через `/setgroup` раньше
  (см. `bot/storage.py: _migrate_legacy_if_needed`). Единственное, что
  сознательно сбрасывается при миграции — незавершённые дуэли/приглашения
  (это временное игровое состояние, не ценные данные; перенос старых
  "зависших" дуэлей приводил к тому, что игроки навсегда считались "занятыми").

## 🔒 Секретная команда владельца — `/creatadd`

Команда для настройки рекламы, которую бот сам рассылает во ВСЕ группы, где
он есть, с заданным интервалом. **Полностью скрыта**: не в меню команд, и
если её вызывает кто-то, кроме владельца (`config.ADMIN_ID`) — бот вообще
никак не отвечает, как будто такой команды не существует.

Поток: `/creatadd` → пришли одним сообщением фото с подписью (текст рекламы)
→ бот спрашивает интервал в днях (можно дробный, например `1.5` или `0.2`)
→ готово. Написать «выкл» вместо фото — отключает текущую рекламу.
См. `bot/handlers/ads.py`.

## ⚠️ Важные допущения (несколько моментов в ТЗ были неоднозначны)

Я принял конкретные решения там, где формулировка допускала разное
прочтение — если что-то не подходит, это буквально несколько чисел/строк
для правки, ищи их по этим меткам:

1. **Размер ставки** — ставка = ВСЕ текущие очки клана игрока на момент
   начала дуэли (снимок сохраняется при выборе количества мин). Выигрыш
   становится новыми очками клана, то есть от боя к бою ставка растёт
   вместе с очками (1000 → 1260 → 1890 и т.д.). Если у клана очков меньше
   `MIN_STAKE_AL` (100, см. `config.py`) — используется этот минимум,
   чтобы дуэль при 0 очках не была бессмысленной.

2. **«Очки клана не прибавляются, а обновляются»** — так как ставка = все
   очки клана, при выигрыше очки клана становятся равны
   `ставка × множитель` (реализовано через `points - stake + stake*mult`,
   что при stake == points на старте даёт то же самое, но остаётся
   корректным и если очки клана успели чуть измениться за время дуэли).
   При проигрыше — `points = points * 0.75`. Смотри `bot/handlers/duel.py`,
   функции `_apply_cashout` / `_apply_loss`.

3. **Определение «победителя дуэли»** для серии побед / `wars_won` — в
   ТЗ описана механика самой игры, но не то, как из двух параллельных
   результатов выбрать «победителя дуэли». Реализовано так: если один
   подорвался, а другой нет — побеждает клан того, кто не подорвался;
   если оба забрали очки — побеждает клан с большим итоговым
   множителем; если оба подорвались — ничья, у обоих сбрасывается серия
   побед. Смотри `_finalize_duel_result` в `bot/handlers/duel.py`.

4. **Матчмейкинг (ротация кланов/игроков)** — правило «аутсайдеры в
   приоритете, но потом обязательно играют и с лидером» и «чтобы играли
   все участники клана по очереди» реализовано в `bot/matchmaking.py`:
   чередование соперника (аутсайдер ↔ лидер по очкам) через флаг в базе,
   и выбор игрока клана с наименьшим числом сыгранных боёв. Это
   осмысленная, но не единственно возможная трактовка — параметры легко
   покрутить в этом файле.

5. **Хранилище** — простая JSON-база (`data/storage.json`) с
   асинхронной блокировкой. Для одной группы этого более чем достаточно;
   если бот будет работать в нескольких группах одновременно, потребуется
   доработка (сейчас `group_chat_id` один на весь бот).

## Сезон войны (30 дней)

Сезон длится `config.SEASON_LENGTH_DAYS` (по умолчанию 30) дней — отсчёт
стартует автоматически при первом запуске бота. Фоновая задача
`bot/season.py: season_watcher_loop` раз в час проверяет, не истёк ли срок;
когда истекает — бот **сам** публикует в боевой чат топ кланов, топ игроков
по лучшему множителю и явно объявленного победителя (клан с наибольшим
количеством очков), после чего очки/серии/победы всех кланов обнуляются и
начинается новый сезон (сами кланы и участники сохраняются). Посмотреть,
сколько дней осталось — команда `/season`. Досрочно подвести итоги вручную
(с подтверждением) — команда `/resetwar` (только владелец бота).

## Новые системы (опыт, репутация, валюта, тактики, магазин)

### Уровни и опыт клана
В конце каждого сезона клан получает ОП = очки_клана_в_конце_сезона × 0.01.
Прогрессия требований к уровню: 20, 30, 40, ... (+10 за каждый следующий
уровень). Титул меняется каждые 3 уровня, значок — по таблице из ТЗ. Уровень,
опыт и достижения клана **не сбрасываются** между сезонами — это его вечная
история. Смотри `bot/leveling.py`.

### Репутация клана
0..1000, растёт за место в сезоне (1 место +100, 2 место +50, остальные +25,
последнее место −25), за пройденные уровни (+3 × номер_уровня) и по-разному
множится достижениями клана. **Обнуляется в начале каждого календарного
года** (сам клан, уровень и достижения — нет). Смотри `bot/reputation.py`.

### Валюта "Те"
Начисляется игроку при победе в раунде = множителю, который он выбил
(x2.19 → +2.19 Те). Тратится в `/shop`. Профиль игрока — `/iam`, либо слово
«Б» в чат, либо «твой б» ответом на чьё-то сообщение.

### Тактики сезона (`/tactic`, выбирает создатель клана)
Пять тактик, подробности в `bot/tactics.py`. Тактика "Тихо не спеша" влияет
только на еженедельное сжатие очков (см. ниже), остальные — на исход
конкретных раундов (штраф за мину, бонус к очкам, валюта).

### Еженедельное сжатие очков
Каждые 7 дней сезона очки всех кланов подтягиваются к среднему на 10%
(формула и множитель — из точного примера в ТЗ: 600/3000/10000 →
993/3153/9453). У клана с тактикой "Тихо не спеша" вместо этого фиксированные
−4% (если выше среднего) / +6% (если ниже).

### Магазин (`/shop`)
Все 7 привилегий из ТЗ реализованы. "Проголосовать за исключение" — это
упрощённое голосование прямо в чате (`/votekick` ответом на сообщение,
нужно 3 голоса «за» в течение 10 минут), а не полноценная система прав.

## Очередь ходов, попытки, закрепление сообщений

- Внутри каждого клана скрыто (не видно пользователям) ведётся очередь
  участников `queue` — вызывают строго по кругу игрок1→игрок2→игрок3→игрок1.
- Если вызванный игрок вообще не сыграл (не выбрал мины) в течение
  `TURN_TIMEOUT_HOURS` (= максимум интервала дуэлей, ~6.5ч) — его попытка
  переходит следующему в очереди с накоплением: 1 пропуск → у следующего
  2 попытки подряд, 2 пропуска → 3 и т.д. Видно только количество оставшихся
  попыток после каждого раунда: "‼️У вас осталось X попыток‼️".
- При пропуске бот пишет в чате `@игрок не пришёл(а) на дуэль⛔ Переход хода
  следующему.` — этой же строкой, что и объявление СЛЕДУЮЩЕЙ дуэли (не
  отдельным сообщением).
- Бот закрепляет сообщение с вызовом на дуэль; как только кто-то открывает
  меню выбора мин — вызов открепляется, закрепляется само меню выбора мин;
  когда дуэль (все попытки обеих сторон) завершена — открепляется. Для этого
  боту нужны права администратора с возможностью закреплять сообщения.

## Недельные события (каждые 7 дней сезона) — их ДВА, разных

1. **Сжатие очков к среднему** — текущий баланс каждого клана НАПРЯМУЮ
   подтягивается к среднему по всем кланам на 10% (формула и множитель —
   из точного примера в ТЗ: 600/3000/10000 → 993.3/3153.3/9453.3 — проверено,
   совпадает).
2. **Накопительный %-бонус к будущим очкам** — ОТДЕЛЬНО от п.1, по месту в
   рейтинге (топ -10%, аутсайдер +10%, промежуточные линейно), складывается
   неделя к неделе и действует как множитель на очки, получаемые В РАУНДАХ
   МИН (не трогает текущий баланс). Тактика "Тихо не спеша" заменяет ТОЛЬКО
   этот пункт на фиксированные ±6% для своего клана — на сжатие (п.1) тактика
   не влияет.

## Тактика — 1 раз за сезон

`/tactic` теперь можно выбрать только один раз за сезон — дальше кнопки
заблокированы до начала следующего. В конце каждого сезона бот сам
напоминает всем владельцам кланов выбрать тактику (в том же сообщении с
итогами), а сразу после создания клана бот тоже подсказывает про `/tactic`.

## ⚠️ Дополнительные допущения по новым системам

- **"Вице"** в описании ачивки «Это... Конец?» я трактовал как покупку
  привилегии "проголосовать за исключение" (vote_kick) — в списке магазина
  нет пункта, буквально названного "вице", это ближайшее по смыслу.
- **Пример "94/100, 6 уровень"** из ТЗ не сходится арифметически с заданной
  прогрессией требований (20, 30, 40, ...) — я реализовал буквально
  прогрессию, а не подогнал её под этот конкретный пример.
- **"Избежать наказания"** и **"следующее поражение не засчитывается"** — это
  РАЗНЫЕ эффекты: первый просто отменяет штраф ×0.75 (раунд всё равно
  считается поражением для статистики/тактик), второй превращает поражение в
  "победу" на x1 (без прибыли, но и без убытка).
- Тактика «Азарт» выбирает случайный эффект **один раз при старте раунда**
  (когда выбрано число мин), а не на каждый клик по полю.
- "Победитель сезона" (ачивка игрока 🎎) выдаётся всем участникам клана,
  занявшего 1 место, в момент подведения итогов сезона.

## Как работает старт дуэли (важный нюанс)

Когда бот объявляет дуэль, **достаточно, чтобы `/minduel` нажал только один
из двух вызванных игроков** — это сразу создаёт игру для обоих сразу (у
каждого своё поле). Второму игроку **не нужно** повторно набирать
`/minduel` — ему нужно просто нажать кнопку с количеством мин 1️⃣–6️⃣ в уже
присланном сообщении с правилами (оно видно всем в группе). Если кто-то по
привычке всё равно наберёт `/minduel` ещё раз, бот теперь не пишет
запутывающее «нет активной дуэли», а объясняет, что делать дальше (нажать
кнопку выбора мин / кликать по своему полю / игра уже завершена).
