# Club Duel Bot — бот войны кланов для Telegram-группы

Телеграм-бот на **aiogram 3** для группового чата: система кланов и
дуэлей-«сапёра» на очки клана.

## Установка и запуск

```bash
python3 -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Токен бота и ID администратора уже вписаны в `config.py` (при желании
вынеси в переменные окружения `BOT_TOKEN` / `ADMIN_ID` — код это уже
поддерживает).

### Первый запуск в группе

1. Добавь бота в группу и дай ему права администратора (нужно для
   редактирования/закрепления сообщений, если захочешь это добавить).
2. Напиши в группе от своего (админского) аккаунта команду:
   ```
   /setgroup
   ```
   Это назначит текущий чат «боевым» — именно туда бот будет раз в день
   (с небольшим случайным разбросом времени) слать объявления о новой дуэли.
3. Участники создают кланы командой `/createclan` (спросит название →
   аватар (или «Пропустить») → девиз) и вступают в существующие кланы
   командой `/join` (карусель ⬅️/➡️ — **одно и то же сообщение
   редактируется**, новых сообщений при листании не создаётся).
4. `/clan` — карточка своего клана: очки, серии побед, участники, лучший
   множитель за бой и т.д.
5. Когда бот сам объявит дуэль (тегнет двух игроков) — один из них жмёт
   `/minduel`, чтобы открыть выбор количества мин и погнать игру.

## Как устроена игра

- Поле 5×5 (25 клеток). Игрок выбирает количество мин (1–6) — чем больше
  мин, тем быстрее растёт множитель выигрыша.
- Оба игрока дуэли играют **на своих отдельных полях параллельно**
  (независимо друг от друга), как и было в ТЗ.
- При открытии безопасной клетки множитель растёт по формуле из ТЗ
  (проверено — совпадает с примерами `x1.01→x1.05→x1.10...` для 1 мины,
  `x1.28→x1.70→x2.30...` для 6 мин).
- Можно нажать «✅ Забрать очки» в любой момент и зафиксировать текущий
  выигрыш, либо продолжать рисковать.
- Если попал на мину — очки **всего клана** умножаются на `0.75`, играть
  дальше нельзя.
- Когда **оба** игрока дуэли закончили свою игру (забрали очки или
  подорвались), бот определяет клан-победитель дуэли, обновляет серию
  побед (`current_win_streak` / `max_win_streak`) и счётчик побед в войне
  кланов (`wars_won`).
- **AFK-защита**: если игрок не делает ход дольше `AFK_TIMEOUT_SECONDS`
  (по умолчанию 3 минуты, см. `config.py`), бот автоматически «забирает»
  его текущий выигрыш за него, чтобы дуэль не зависала — это и есть
  «переход хода при АФК» из ТЗ.

## ⚠️ Важные допущения (несколько моментов в ТЗ были неоднозначны)

Я принял конкретные решения там, где формулировка допускала разное
прочтение — если что-то не подходит, это буквально несколько чисел/строк
для правки, ищи их по этим меткам:

1. **Размер ставки** — ставка = ВСЕ текущие очки клана игрока на момент
   начала дуэли (снимок сохраняется при выборе количества мин). Выигрыш
   становится новыми очками клана, то есть от боя к бою ставка растёт
   вместе с очками (1000 → 1260 → 1890 и т.д.). Если у клана очков меньше
   `MIN_STAKE_AL` (100, см. `config.py`) — используется этот минимум,
   чтобы дуэль при 0 очках не была бессмысленной.

2. **«Очки клана не прибавляются, а обновляются»** — так как ставка = все
   очки клана, при выигрыше очки клана становятся равны
   `ставка × множитель` (реализовано через `points - stake + stake*mult`,
   что при stake == points на старте даёт то же самое, но остаётся
   корректным и если очки клана успели чуть измениться за время дуэли).
   При проигрыше — `points = points * 0.75`. Смотри `bot/handlers/duel.py`,
   функции `_apply_cashout` / `_apply_loss`.

3. **Определение «победителя дуэли»** для серии побед / `wars_won` — в
   ТЗ описана механика самой игры, но не то, как из двух параллельных
   результатов выбрать «победителя дуэли». Реализовано так: если один
   подорвался, а другой нет — побеждает клан того, кто не подорвался;
   если оба забрали очки — побеждает клан с большим итоговым
   множителем; если оба подорвались — ничья, у обоих сбрасывается серия
   побед. Смотри `_finalize_duel_result` в `bot/handlers/duel.py`.

4. **Матчмейкинг (ротация кланов/игроков)** — правило «аутсайдеры в
   приоритете, но потом обязательно играют и с лидером» и «чтобы играли
   все участники клана по очереди» реализовано в `bot/matchmaking.py`:
   чередование соперника (аутсайдер ↔ лидер по очкам) через флаг в базе,
   и выбор игрока клана с наименьшим числом сыгранных боёв. Это
   осмысленная, но не единственно возможная трактовка — параметры легко
   покрутить в этом файле.

5. **Хранилище** — простая JSON-база (`data/storage.json`) с
   асинхронной блокировкой. Для одной группы этого более чем достаточно;
   если бот будет работать в нескольких группах одновременно, потребуется
   доработка (сейчас `group_chat_id` один на весь бот).

## Структура проекта

```
club_duel_bot/
├── main.py                    # точка входа, запуск polling + фоновых задач
├── config.py                  # токен, id админа, все игровые константы
├── requirements.txt
├── data/
│   └── storage.json           # создаётся автоматически при первом запуске
└── bot/
    ├── storage.py             # JSON-хранилище (async, с блокировкой)
    ├── game.py                # математика множителей + генерация мин
    ├── texts.py                # шаблоны текстов (HTML/<blockquote>)
    ├── keyboards.py           # инлайн-клавиатуры
    ├── matchmaking.py         # подбор пар кланов/игроков + планировщик "раз в день"
    ├── leaderboard.py         # текст таблицы лидеров (общий для /top и итогов сезона)
    ├── season.py              # автоматическое завершение сезона через 30 дней
    └── handlers/
        ├── clan_create.py     # /createclan (FSM)
        ├── clan_join.py       # /join (карусель, редактирование одного сообщения)
        ├── clan_info.py       # /clan, /setgroup (админ)
        ├── clan_manage.py     # /deleteclan, /kick, /leaveclan, /top, /resetwar
        └── duel.py            # /minduel + вся игровая логика + AFK-вотчер
```

## Команды

| Команда | Кто | Описание |
|---|---|---|
| `/createclan` | любой | создать клан |
| `/join` | любой | карусель кланов, вступить |
| `/clan` | любой | карточка своего клана |
| `/minduel` | вызванный игрок | начать назначенную дуэль |
| `/leaveclan` | любой | покинуть свой клан (кроме создателя) |
| `/kick` | создатель клана | исключить участника (можно ответом на его сообщение) |
| `/deleteclan` | создатель клана | расформировать клан (с подтверждением) |
| `/top` | любой | топ кланов по очкам + топ игроков по лучшему множителю |
| `/season` | любой | сколько дней прошло/осталось до конца текущего сезона |
| `/setgroup` | только админ (ID из `config.py`) | назначить текущий чат «боевым» |
| `/forceduel` | только админ | объявить дуэль прямо сейчас (можно из ЛС боту), не дожидаясь расписания |
| `/resetwar` | только админ | досрочно подвести итоги сезона и начать новый (обычно это происходит автоматически) |

## Сезон войны (30 дней)

Сезон длится `config.SEASON_LENGTH_DAYS` (по умолчанию 30) дней — отсчёт
стартует автоматически при первом запуске бота. Фоновая задача
`bot/season.py: season_watcher_loop` раз в час проверяет, не истёк ли срок;
когда истекает — бот **сам** публикует в боевой чат топ кланов, топ игроков
по лучшему множителю и явно объявленного победителя (клан с наибольшим
количеством очков), после чего очки/серии/победы всех кланов обнуляются и
начинается новый сезон (сами кланы и участники сохраняются). Посмотреть,
сколько дней осталось — команда `/season`. Досрочно подвести итоги вручную
(с подтверждением) — команда `/resetwar` (только владелец бота).

## Как работает старт дуэли (важный нюанс)

Когда бот объявляет дуэль, **достаточно, чтобы `/minduel` нажал только один
из двух вызванных игроков** — это сразу создаёт игру для обоих сразу (у
каждого своё поле). Второму игроку **не нужно** повторно набирать
`/minduel` — ему нужно просто нажать кнопку с количеством мин 1️⃣–6️⃣ в уже
присланном сообщении с правилами (оно видно всем в группе). Если кто-то по
привычке всё равно наберёт `/minduel` ещё раз, бот теперь не пишет
запутывающее «нет активной дуэли», а объясняет, что делать дальше (нажать
кнопку выбора мин / кликать по своему полю / игра уже завершена).
